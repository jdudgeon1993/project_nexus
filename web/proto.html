Design notes / things to play with

Header

  • Collapsed to one slim row: back, route pill, title+meta, search icon.
  • Search expands inline below header instead of always-visible bar.

Timeline (main view)

  • Each stop card now carries: countdown + clock time, live status (at platform / in transit / delay), "then X, Y min" for following trains, transfer badges, accessibility icon, skipped indicator.
  • Color of countdown = on-time/late/early (reuses formatDelay logic).
  • "Train here" stop gets a glowing dot on the timeline + highlighted marker.

Transfer badges

  • Real route colors from transfersByStop — circle for rail, rounded rect for bus.
  • S/D/A/P/S placeholders → actual connecting routes (e.g. B, G lines + bus routes at Union Station).

FAB

  • Replaces the Settings/Plan/Alerts/Directions/Schedule tab row entirely.
  • Tap "+" → expands a vertical stack of pill buttons, tap again collapses.
  • Alerts item shows count + amber accent when active alerts exist.
  • Each item would open its existing panel as an overlay/sheet rather than a tab swap, so the timeline stays mounted underneath.

Open questions

  • Overlay vs. replace-content when a FAB item is tapped — overlay (drawer from bottom) probably keeps "everything feels like one page."
  • Direction toggle row — keep as-is, or fold into header as a dropdown?
  • Live vehicle markers — currently a separate list; here they're folded into stop rows. Do we still want a "Train 1 / Train 2" summary somewhere, or is per-stop status enough?
